# Intake Questions

Use this question bank at the start of every new short-drama/comic-drama project. Ask in batches and adapt to the user's answers. Do not dump every question at once.

## Round 1 - Foundation

Ask 8-12 of these:

1. 你这次想做短剧、短漫剧、漫画旁白视频，还是真人/动画感短片？
2. 你的一句话脑洞是什么？如果还没有，我可以先问你几个方向题。
3. 目标观众是谁：男频、女频、宝妈、职场人、学生、二次元、商业人群，还是泛娱乐？
4. 你最想让观众爽在哪里：逆袭、打脸、复仇、甜宠、悬疑反转、恐怖刺激、搞笑、治愈、知识爽感？
5. 主角是谁？他/她一开始缺什么、怕什么、被谁压迫？
6. 反派或压力源是谁？是人、制度、怪物、家庭、公司、系统，还是误会？
7. 你想做几集？单集大概多少秒？要不要先做 1 集样片？
8. 画面比例和平台是什么？默认可用 9:16。
9. 想要什么视觉风格：3D CG、国漫、日漫、厚涂漫画、真人电影感、赛博、废土、古风、都市？
10. 有没有参考作品、参考图、参考视频，或者明确不要像什么？
11. 输出要到哪一步：只要剧本、要分镜图、要小云雀生成视频，还是完整流程？
12. 这次可以消耗小云雀积分/API 吗？还是先只做可审稿的制作包？

## Round 2 - Pressure Test

Ask these only after Round 1 when needed:

- 这个题材的前三秒钩子是什么？观众为什么不划走？
- 主角的反常识点是什么？为什么他/她会让观众想继续看？
- 每集结尾的卡点是悬念、爽点、反转，还是情绪爆发？
- 哪些角色或场景必须保持一致，不能生成跑偏？
- 是否需要旁白型剧本？是否要保留对白？
- 是否有平台敏感内容、血腥、未成年人、政治、真实名人、品牌/IP 风险？
- 你更在意速度、成本、画面质量，还是角色一致性？
- 如果只能先做一个样片，你最想验证剧情、画风、角色，还是视频生成稳定性？

## Confirmation Summary

Before creating deliverables, summarize:

- project format and target platform
- audience and emotional promise
- episode count and duration
- visual style and ratio
- protagonist/antagonist/core contradiction
- output scope: script, storyboard images through `$imagegen`, Xiaoyunque video, or all
- assumptions and unresolved risks

Then ask for confirmation. Start production only after confirmation, unless the user explicitly says to proceed without confirmation.
